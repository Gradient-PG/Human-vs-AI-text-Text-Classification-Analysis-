The pipeline is a method-agnostic CV fold loop driven by raid_analysis/experiments/runner.py. Every experiment plugs into the same loop by supplying a selector (how neurons are chosen) and an evaluator (how the chosen set is scored). The primary experiment (sparse_probe) performs selection; dependent experiments (ablation, patching, confound) reuse its per-fold selections without re-doing selection, via source_loader.load_source_selections.

Per fold the runner strictly enforces a three-step train/test separation:

  1. Step 1 — Selection: either selector.select(train_acts, train_labels) or a precomputed SelectionResult is loaded. Selection never sees test data.
  2. Step 2 — Evaluation probe: a fresh L2 logistic regression (train_eval_probe) is fit on the full train fold using all features. The eval probe is the fixed measurement instrument; it is not retrained per evaluator.
  3. Step 3 — Evaluation: evaluator.evaluate(test_acts, test_labels, test_meta, selection, eval_probe). Evaluators see test data and the frozen artifacts from steps 1–2.

2.2 Data & Splits

  • Activations: concatenation of all transformer layers, loaded per-generator from results/activations_raid_<slug>/. Global feature axis has D = n_layers × hidden_dim. For BERT-base this is 12 × 768 = 9,216.
  • Balanced subsampling: _balanced_subsample in data/loading.py draws max_samples / 2 per class with seed 42. Headline run uses max_samples = 7,500 — bumped from 5,000 because the multi-generator stability sweep (§4.5) showed Jaccard rising another +0.07 to +0.11 between N = 5,000 and N = 7,500 (largest gain on mistral) and the cross-generator Jaccard spread tightening from ~7 pp to ~2.7 pp. N = 10,000 was not chosen because the available pool caps K = 1, so stability there is unmeasurable.
  • Splits: StratifiedKFold with a label × domain stratification key. Each fold has balanced class and domain representation, which is required for same-domain patching pairs and domain-level confound diagnostics.
  • Metadata (SampleMetadata): text_lengths, domain_ids, domain_names. There is no explicit “generator” field — generator identity is implicit in which activation directory is loaded.
  • Splits are persisted as splits.json per experiment and reused across dependent experiments for the same run_id.

2.3 Selection: SparseProbeSelector (L1)

Implementation: raid_analysis/selection/sparse_probe.py.

Fixed-C selection (current default)

  1. Fit L1 logistic regression (penalty='l1', solver='liblinear', max_iter=1000) on the full outer train fold at C = 0.005.
  2. Report selected neurons (non-zero coefficients) and a ranking by descending |coef|.

Selector C and the headline N were chosen jointly via the multi-generator stability sweep in §4.5: C = 0.005 at N = 7,500 gives the highest pairwise Jaccard (0.72–0.75 across gpt4 / llama-chat / mistral; min-over-3-gen 0.724) among the C values we tested, while remaining cheap to fit (~30 s per fold). The previous “knee detection over a 5-point C sweep” pipeline is still implemented but bypassed (single-element C_values list short-circuits it).

Solver change Switched from saga (stochastic average gradient) to liblinear (coordinate descent). Both produce valid L1 solutions, but liblinear is dramatically more aggressive at producing exact zeros on this kind of high-dimensional, underdetermined problem — e.g. on GPT-4 at the same C, saga at max_iter=5000 retained 167 features per fold while liblinear at max_iter=1000 retains 74.1 ± 2.9 (at N = 7,500) with negligible accuracy difference. Speed improvement is ~10–100× per fold. The earlier 167-feature numbers are an artifact of saga's slower convergence on L1 sparsity, not a different signal.

SelectionResult payload (persisted per fold)

  • neuron_indices: set of (layer, neuron) tuples with non-zero coefficients.
  • ranking: (layer, neuron) ordered by descending |coef| (used as top-k order in ablation and patching).
  • probe, scaler: the fitted L1 probe and its StandardScaler (used only for the diagnostic selector_test_accuracy).
  • train_mean: per-dimension training-fold mean (used by mean ablation to avoid test leakage).
  • metadata: C, solver, n_nonzero, n_features, train_accuracy, single-point sweep_curve.

2.4 Evaluation Probe (L2)

Implementation: raid_analysis/evaluation/probe_factory.py.

  • StandardScaler fit on the full train fold activations.
  • LogisticRegression(C=eval_probe_C, solver='lbfgs', ...) — L2 default, all D features.
  • Serialized per fold as l2_probe_weights.npy, l2_probe_intercept.npy, l2_scaler_mean.npy, l2_scaler_scale.npy.
  • The L2 probe is independent of what the selector chose. This is the critical design fact that shapes ablation interpretation (see §4.1).

2.6 Dependent Experiments

Ablation, patching, and confound all set source_experiment: sparse_probe in YAML. The runner:

  1. Resolves source_dir to results/experiments/<run_id>/sparse_probe/<generator>.
  2. Loads SelectionResult per fold via load_source_selections (reads selection.json + train_mean.npy).
  3. Passes these as precomputed_selections; Step 1 is skipped. Step 2 (L2 probe) is still fit fresh from the current config.

Reuse ensures ablation/patching/confound operate on exactly the same neuron sets, rankings, and splits as the primary sparse_probe run.

Ablation evaluator (evaluation/ablation.py)

  • For each k in k_values, take top-k global indices from selection.ranking.
  • Mean-ablate those columns in test activations: replace with selection.train_mean[idx]. Score L2 eval probe.
  • Random-k baseline: draw n_random_seeds random k-subsets (seed 42 — this is deterministic across folds; see §4.3).
  • Report per-k: selected_acc, selected_drop, random_acc_mean/std, random_drop_mean/std.

Patching evaluator (evaluation/patching.py)

  • Restrict to human test samples (label==0) that have at least one same-domain AI sample in the test fold (valid_humans).
  • Compute base_preds = eval_probe.predict(base_acts) once.
  • For each of n_shuffles random donor assignments (permuted within domain), copy donor_activations[:, top_k_indices] into base_acts[:, top_k_indices]. Score eval_probe.predict(patched). A “flip” is human (0) → AI (1).
  • Flip rate per shuffle = flips / |valid_humans|. Mean/std over shuffles.
  • Random-k baseline: n_random_draws random neuron subsets; for each, run n_shuffles pairings and report mean per-draw, then mean/std across draws.
  • Per-domain breakdown at the largest k used.

Confound evaluator (evaluation/confound.py, to be removed)

  • For each selected neuron, on the pooled test fold: Spearman(act, text_length) and one-way ANOVA(act, domain_id).
  • “Flagged” if p < 0.05 under either test. No multiple-comparison correction. Tests are on pooled human+AI samples, which conflates class signal with domain association (see §4.2).

Three regimes, not one. The clean “univariate needs more features” story from the old report only describes 3/5 generators in the new panel:

  • Parsimony regime (gpt4, mistral, llama): L1 picks 4–7× fewer features than AUC and the L1 ablation drop is smaller. L1 is strictly more parsimonious.
  • Convergence regime (gpt2): AUC and L1 pick the same number of features (~85) and produce indistinguishable ablation effects (both ~0.001). Univariate is sufficient here. Consistent with gpt2 being the difficulty mid-point, where the multivariate advantage of L1 vanishes because the per-axis signal is already strong enough.
  • Reversal regime (cohere-chat): AUC picks roughly half the features L1 does (38 vs 72), with a smaller ablation drop. For cohere, top-AUC features are the more efficient encoding. Consistent with §3.2 (the L1 set on cohere has a real necessity contribution — ablating it costs more than ablating an AUC-equivalent set).

Methods-claim status: the “univariate vs multivariate” claim is now generator-dependent, not universal. The cleanest framing for the paper is parsimony on the “normal” generators with explicit acknowledgement of the gpt2 convergence and cohere reversal as evidence that the dominance of multivariate L1 is not unconditional. 4. Methodological Findings

4.1 Redundancy story (ablation ↔ patching)

“Ablation is weak but patching works” is not a contradiction. Ablation tests necessity against a probe with 9,216 features — removing ≤4% is compensated by redundant correlates. Patching tests sufficiency by injecting AI-class signal into human rows — wrong-class signal in selected features competes with human signal elsewhere. Both confirm a redundantly distributed representation (§3.2, §3.3). The restricted probe (§3.7) adds a presentation sentence but no new scientific content.

4.2 Per-neuron confound test — why we are dropping it

The test pools human+AI and runs per-neuron marginal statistics. A genuine detector whose activation varies across domains because AI text differs per domain will be flagged identically to a confound. 99% flagged, so it discriminates nothing. Replacement: leave-one-generator-out generalization (§5.1 A). Domain confounds are addressed methodologically via StratifiedKFold on (label × domain) and same-domain donor sampling in the patching experiment; a dedicated domain probe is not required.

4.3 Which neurons do ablation/patching actually intervene on?

Each fold’s ablation/patching operates on that fold’s L1-selected set, not the aggregated stable set. Using the stable set would leak cross-fold information (stability depends on overlapping training data). Consequence: results strictly say “the per-fold selected set is sufficient/not-necessary,” not “the stable core drives detection.” A future appendix experiment could intervene on the stable set directly to close this gap (~30 LoC).

4.4 Selection vs evaluation separation (minor notes)

  • L1 probe is used only for the diagnostic selector_test_accuracy; all evaluation uses a fresh L2 probe. Good separation.
  • Ablation/patching random-k baselines use a fixed RandomState(42) per call — correlated across the sweep. Not wrong but worth fixing (derive seed from fold/seed/k).

4.5 Sample size & C selection via multi-generator stability sweep

Why we abandoned knee detection. The original pipeline picked C per-fold via a knee in the (n_nonzero, val_accuracy) curve over 5 C values. This conflated three sources of variation: CV-fold variance, knee-detector geometry, and sample size. In particular, runs at small N would land on degenerate sweep curves (anchor point with 0 features at C = 0.001), pulling the knee toward dense solutions and creating spurious “sample-size effects” that were really artefacts of the knee anchor.

Replacement methodology. Run a fixed C at each N and measure selection stability directly via independent stratified subsamples, repeated across multiple generators. For each (generator, N, C):

  1. Draw K = 15 independent stratified subsamples of size N from the 10k pool (balanced across class × domain).
  2. For each draw, fit one L1 logistic regression at the fixed C; record the selected neuron set S_k.
  3. Compute pairwise Jaccard similarity over {S_1, …, S_K} (K choose 2 = 105 pairs); report mean as the stability score.
  4. Compute the core stable set: neurons selected in ≥ 80% of the K draws.

Source of the discrepancy is unclear; the paper may have used an intermediate computation or a different metric. Fix: replace the Smaller model column with the correct values above.

Issue 2 (wrong number) — “four of five” should be “five of six” in §4.2

After MPT was added as the sixth generator, the ablation table (tab:ablation-full) was correctly extended to six rows, but the body text of §4.2 still reads: “For four of five generators the result is negative.” The correct count is five of six (GPT-4, GPT-2, MPT, Mistral, LLaMA are all within noise; Cohere is the exception). The figure caption and conclusion already say “5 of 6” correctly. Fix: change “four of five” to “five of six” in the body text.

Issue 3 (inconsistency / overstatement) — Patching ratio “10–23×” vs “10–16×” within paper

The abstract and intro §1 ¶4 say “10–23× above a random-feature baseline”. The contributions bullet and the figure caption (fig:pipeline) say “10–16×”. The 23× figure comes from Mistral when ratios are computed per-cell and averaged: some cells have near-zero random baselines (< 0.03%), inflating the per-cell ratio to 55–74× and driving the mean to 23.5×. The ratio of overall means for Mistral is 1.73% ÷ 0.12% = 14.4×. The more robust “10–16×” figure (used in contributions and figure) is more representative. Fix: standardise on “10–16×” in abstract and §1 ¶4, and add a footnote noting the denominator instability.

Issue 4 (misinterpretation) — LOGO Table 6 AUC column contains Full L2 AUC, not Sel L2 AUC

The LOGO table (tab:logo) has a column labelled only “AUC” containing 0.946, 0.993, 0.999, 0.982, 0.995. These match the Full L2 AUC values from summary.json exactly. The caption text correctly states “Sel L2 AUC ranges from 0.863 (cohere) to 0.986 (llama)” as a separate sentence, but a reader scanning the table will assume the column shows Sel L2 AUC. The actual Sel L2 AUC values are: cohere 0.863, gpt 0.953, llama 0.986, mistral 0.925, mpt 0.926 — notably lower and more informative for assessing transfer quality. Fix: rename the column to “Full AUC” and add a “Sel AUC” column with the correct values.

Issue 5 (overstatement) — Abstract claims stable neurons “achieve 90.5–98.8% accuracy”

The abstract says “stable sets of 45–62 neurons…that achieve 90.5–98.8% detection accuracy”. The 90.5–98.8% range is the full-feature L2 probe accuracy (all 9,216 features), not the accuracy of a probe restricted to the stable neurons. The actual restricted-probe (smaller model) accuracy is 86.5–97.2% (Cohere 86.5%, GPT-4 97.2%). The §3.1 prose correctly uses “yet the L2 evaluation probe achieves…”, making clear the full-feature probe is being measured, but the abstract phrasing implies the stable neurons alone achieve the stated accuracy. Fix: rewrite abstract to say “…are stable; a full-feature L2 probe achieves 90.5–98.8% accuracy, and a probe restricted to the stable set retains 86.5–97.2%”, or similar.

Issue 6 (overstatement) — §3.1 says “0.5–0.9%” for stable set size but upper bound is ~0.7%

The text says “Between 45 and 62 neurons — 0.5–0.9% of the 9,216-feature CLS representation”. Actual fractions: 45/9216 = 0.49% and 62/9216 = 0.67%. The upper bound of 0.9% overstates the stable set size; it appears to come from the per-cell selected count (GPT-2 mean 84.7/9216 = 0.92%), which is a different quantity. Fix: change to “0.5–0.7%” for stable sets, or clarify parenthetically that the figure refers to the per-cell selected count.
